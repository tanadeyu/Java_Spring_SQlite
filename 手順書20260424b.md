## 2. DB設計

### 2.1 ER図

```
┌─────────────────┐
│    customers    │ 顧客マスタ
└────────┬────────┘
         │ 1:N
         ▼
┌─────────────────┐
│  sale_headers   │ 販売ヘッダー
└────────┬────────┘
         │ 1:N
         ▼
┌─────────────────┐
│     sales       │ 販売明細
└────────┬────────┘
         │ N:1
         ▼
┌─────────────────┐           ┌─────────────────┐
│    products     │ 商品マスタ │   categories    │ カテゴリマスタ
│                 │    N:N    │                 │
└─────────────────┘──────────▶└─────────────────┘
```

### 2.2 テーブル定義

**customers（顧客マスタ）**
| カラム名 | 型 | 説明 |
|---------|---|------|
| id | LONG | 主キー（自動採番） |
| name | STRING | 氏名 |
| email | STRING | メールアドレス（UNIQUE） |
| member_number | STRING | 会員番号（UNIQUE） |

**products（商品マスタ）**
| カラム名 | 型 | 説明 |
|---------|---|------|
| id | LONG | 主キー |
| name | STRING | 商品名 |
| description | STRING | 説明 |
| price | INTEGER | 価格 |
| stock | INTEGER | 在庫数（デフォルト値: 0） |
| publishedDate | LOCAL_DATE | 発行日（書籍の場合など） |
| categories | SET<Category> | カテゴリ（N:N） |

**categories（カテゴリマスタ）**
| カラム名 | 型 | 説明 |
|---------|---|------|
| id | LONG | 主キー |
| name | STRING | カテゴリ名（UNIQUE） |

**sale_headers（販売ヘッダー）**
| カラム名 | 型 | 説明 |
|---------|---|------|
| id | LONG | 主キー |
| customer | Customer | 顧客（FK） |
| saleDate | LOCAL_DATE | 販売日 |
| totalAmount | INTEGER | 合計金額 |

**sales（販売明細）**
| カラム名 | 型 | 説明 |
|---------|---|------|
| id | LONG | 主キー |
| header | SaleHeader | ヘッダー（FK） |
| product | Product | 商品（FK） |
| quantity | INTEGER | 数量 |
| salePrice | INTEGER | 販売単価 |

## 3. 実装手順

### Step 1: EclipseでSpring Bootプロジェクト作成

1. ファイル → 新規 → その他 → 「Spring スターター・プロジェクト」
2. 設定:
   - Name: `bookstore`
   - Type: Maven
   - Group: `com.example`
   - Artifact: `bookstore`
   - Java Version: 21
   - Boot Version: **3.3.4**
3. 依存関係: Spring Web, Spring Data JPA, Thymeleaf, Validation
4. **⚠️ H2 Databaseは追加しないでください**

### Step 2: SQLite依存の追加

**pom.xml** に以下を追加:
```xml
<dependency>
    <groupId>org.xerial</groupId>
    <artifactId>sqlite-jdbc</artifactId>
    <version>3.46.0.0</version>
</dependency>
<dependency>
    <groupId>org.hibernate.orm</groupId>
    <artifactId>hibernate-community-dialects</artifactId>
</dependency>
```

**application.properties** を設定:
```properties
spring.datasource.url=jdbc:sqlite:bookstore.db
spring.datasource.driver-class-name=org.sqlite.JDBC
spring.jpa.database-platform=org.hibernate.community.dialect.SQLiteDialect
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
spring.web.locale=ja_JP
```

### Step 3: エンティティ作成

**ファイル**: `src/main/java/com/example/bookstore/entity/`

| クラス | 説明 |
|--------|------|
| Category.java | カテゴリエンティティ（@Entity, @ManyToMany） |
| Product.java | 商品エンティティ（categories, salesとの関連） |
| Customer.java | 顧客エンティティ（memberNumberフィールド） |
| SaleHeader.java | 販売ヘッダー（customer, details） |
| Sale.java | 販売明細（header, product, quantity, salePrice） |

### Step 4: リポジトリ作成

**ファイル**: `src/main/java/com/example/bookstore/repository/`

| インターフェース | 説明 |
|----------------|------|
| CategoryRepository | `findByName(String)` |
| ProductRepository | `findByNameContaining`, `findByStockLessThan` |
| CustomerRepository | `findByEmail`, `findByMemberNumber` |
| SaleHeaderRepository | 販売ヘッダーのCRUD |
| **SaleRepository** | **分析クエリ（@Query使用）** |

**SaleRepositoryの分析クエリ**:
- 売上ランキング（GROUP BY, ORDER BY）
- カテゴリ別集計（JOIN, GROUP BY）
- 月次売上推移（日付関数）
- 顧客別売上

### Step 5: サービス作成

**DTO** (`src/main/java/com/example/bookstore/dto/`):
- ProductRanking（商品名、売上金額、販売回数）
- CategorySales（カテゴリ名、売上金額、商品数）
- MonthlySales（月、売上金額、販売回数）
- CustomerSales（顧客名、総購入金額、購入回数）

**サービス** (`src/main/java/com/example/bookstore/service/`):
- ProductService（商品管理）
- CustomerService（顧客管理）
- SaleHeaderService（販売ヘッダー管理）
- **SaleService**（分析機能: ランキング、カテゴリ集計、月次、顧客別）

### Step 6: コントローラー作成

**ファイル**: `src/main/java/com/example/bookstore/controller/`

| クラス | URL | 説明 |
|--------|-----|------|
| MainController | / | ダッシュボード（分析結果表示） |
| ProductController | /products | 商品一覧（閲覧のみ） |
| CustomerController | /customers | 顧客一覧（閲覧のみ） |
| SaleHeaderController | /sales | 販売一覧、登録 |

### Step 7: ビュー作成

**ファイル**: `src/main/resources/templates/`

| ファイル | 説明 |
|---------|------|
| index.html | ダッシュボード（ランキング、カテゴリ別、月次、顧客別） |
| products.html | 商品一覧（ID、名前、説明、価格、在庫、カテゴリ） |
| customers.html | 顧客一覧（ID、氏名、メール、会員番号） |
| sales.html | 販売一覧（ヘッダー一覧、詳細モーダル） |
| sales-form.html | 販売登録（ヘッダー+明細1〜5行） |

### Step 8: データ初期化

**ファイル**: `DataInitializer.java`（CommandLineRunner実装）

サンプルデータを自動生成:
- カテゴリ: 5種類（小説、ビジネス書、コミック、雑誌、教育・参考書）
- 商品: 10件（書籍関連）
- 顧客: 5件（会員番号: MEM00000001〜）
- 販売ヘッダー: 15件
- 販売明細: 30件以上

## 5. トラブルシューティング

### Q1: pom.xmlに「cvc-elt.1.a」エラー

**→ 無視して問題ありません**。実行に影響しません。

### Q2: Spring Boot 4.xしか選べない

**→ 手動で 3.3.4 を入力してください**。

**→ いいえ、SQLiteのみ使用します**。H2依存は削除してください。
