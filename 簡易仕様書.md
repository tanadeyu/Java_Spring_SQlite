## 1. システム概要

### 1.1 目的
商品販売データを収集・分析し、売上状況を可視化するシステムを構築する。

### 1.2 対象ユーザー
- 小売業の店舗管理者
- 販売データの分析担当者
- 在庫管理担当者

### 1.3 想定運用環境
- Webブラウザ（最新版）
- ローカル環境または社内LAN

---

## 2. 機能一覧

| 機能ID | 機能名 | 概要 | 優先度 |
|--------|--------|------|--------|
| F01 | 商品管理 | 商品マスタの閲覧 | 高 |
| F02 | 顧客管理 | 顧客マスタの閲覧 | 高 |
| F03 | 販売管理 | 販売データの登録・閲覧（ヘッダーー明細構造） | 高 |
| F04 | 売上ランキング | 商品別売上順位の表示 | 中 |
| F05 | カテゴリ別集計 | カテゴリごとの売上集計 | 中 |
| F06 | 顧客別分析 | 顧客ごとの購入状況表示 | 低 |

### 3.2 テーブル定義

#### 3.2.1 customers（顧客マスタ）

| カラム名 | データ型 | 必須 | 説明 |
|---------|---------|------|------|
| id | BIGINT | ○ | 主キー（自動採番） |
| name | VARCHAR | ○ | 氏名 |
| email | VARCHAR | ○ | メールアドレス（ユニーク） |
| member_number | VARCHAR | - | 会員番号（ユニーク） |

#### 3.2.2 products（商品マスタ）

| カラム名 | データ型 | 必須 | 説明 |
|---------|---------|------|------|
| id | BIGINT | ○ | 主キー（自動採番） |
| name | VARCHAR | ○ | 商品名 |
| description | VARCHAR | - | 説明 |
| price | INTEGER | - | 価格 |
| stock | INTEGER | - | 在庫数（初期値0） |
| published_date | DATE | - | 発行日 |

#### 3.2.3 categories（カテゴリマスタ）

| カラム名 | データ型 | 必須 | 説明 |
|---------|---------|------|------|
| id | BIGINT | ○ | 主キー（自動採番） |
| name | VARCHAR | ○ | カテゴリ名（ユニーク） |

#### 3.2.4 sale_headers（販売ヘッダー）

| カラム名 | データ型 | 必須 | 説明 |
|---------|---------|------|------|
| id | BIGINT | ○ | 主キー（自動採番） |
| customer_id | BIGINT | ○ | 顧客ID（外部キー） |
| sale_date | DATE | ○ | 販売日 |
| total_amount | INTEGER | ○ | 合計金額 |

#### 3.2.5 sales（販売明細）

| カラム名 | データ型 | 必須 | 説明 |
|---------|---------|------|------|
| id | BIGINT | ○ | 主キー（自動採番） |
| header_id | BIGINT | ○ | ヘッダーID（外部キー） |
| product_id | BIGINT | ○ | 商品ID（外部キー） |
| quantity | INTEGER | ○ | 数量 |
| sale_price | INTEGER | ○ | 販売単価 |

#### 3.2.6 product_categories（商品-カテゴリ中間テーブル）

| カラム名 | データ型 | 必須 | 説明 |
|---------|---------|------|------|
| product_id | BIGINT | ○ | 商品ID（外部キー） |
| category_id | BIGINT | ○ | カテゴリID（外部キー） |

### 3.3 リレーション定義

| 親テーブル | 子テーブル | 関係 | 説明 |
|-----------|-----------|------|------|
| customers | sale_headers | 1:N | 1顧客 : 複数購入 |
| sale_headers | sales | 1:N | 1伝票 : 複数明細 |
| products | sales | 1:N | 1商品 : 複数明細 |
| products | categories | N:N | 商品 : カテゴリ（多対多） |

## 4. 機能詳細

### 4.1 商品管理（F01）

#### 4.1.1 機能概要

#### 4.1.2 表示項目

| 項目 | 説明 |
|------|------|
| ID | 商品ID |
| 商品名 | |
| 説明 | |
| 価格 | |
| 在庫 | 在庫数 |
| カテゴリ | 複数カテゴリ |

### 4.2 顧客管理（F02）

#### 4.2.1 機能概要
顧客情報の一覧表示を行う。データはシステム初期化時に自動生成される。

#### 4.2.2 表示項目

| 項目 | 説明 |
|------|------|
| ID | 顧客ID |
| 氏名 | |
| メールアドレス | |
| 会員番号 | |

### 4.3 販売管理（F03）

#### 4.3.1 機能概要
販売データの登録と閲覧を行う。**ヘッダーー明細構造**を採用し、1回の購入で複数商品を登録可能。

#### 4.3.2 構造
- **ヘッダー**: 顧客、販売日、合計金額
- **明細**: 商品、数量、販売単価（最大5件）

#### 4.3.3 入力項目

**ヘッダー情報**
| 項目 | 必須 | 説明 |
|------|------|------|
| 顧客 | ○ | 既存顧客から選択 |
| 販売日 | ○ | デフォルト当日 |

**明細情報（最大5件）**
| 項目 | 必須 | 説明 |
|------|------|------|
| 商品 | ○ | 既存商品から選択（価格表示） |
| 数量 | ○ | 1～999 |

#### 4.3.4 機能要件
- 1回の購入で最大5商品まで登録可能
- 販売単価は商品マスターの価格を自動使用
- 合計金額は自動計算

### 4.4 売上ランキング（F04）

#### 4.4.1 機能概要
商品別の売上順位を表示する。

#### 4.4.2 表示項目

| 項目 | 説明 |
|------|------|
| 順位 | 売上金額順 |
| 商品名 | |
| 売上金額 | 合計金額 |
| 販売回数 | |

### 4.5 カテゴリ別集計（F05）

#### 4.5.1 機能概要
カテゴリごとの売上を集計表示する。

#### 4.5.2 表示項目

| 項目 | 説明 |
|------|------|
| カテゴリ名 | |
| 売上金額 | 合計金額 |
| 商品数 | カテゴリ内の商品種類 |

### 4.6 顧客別分析（F07）

#### 4.7.1 機能概要
顧客ごとの購入状況を表示する。

#### 4.7.2 表示項目

| 項目 | 説明 |
|------|------|
| 顧客名 | |
| 総購入金額 | 合計金額 |
| 購入回数 | 合計回数 |

## 5. 画面設計

### 5.1 画面一覧

| 画面ID | 画面名 | 概要 |
|--------|--------|------|
| P01 | ダッシュボード | 分析結果の一覧表示 |
| P02 | 商品一覧 | 商品マスタの一覧表示 |
| P03 | 顧客一覧 | 顧客マスタの一覧表示 |

| P05 | 販売登録 | 販売データの入力（最大5商品） |

### 5.2 販売一覧（P04）

#### 5.2.1 表示項目

| 項目 | 説明 |
|------|------|
| ID | ヘッダーID |
| 日付 | 販売日 |
| 顧客 | 顧客名 |
| 合計金額 | 明細の合計 |
| 操作 | 詳細・削除 |

#### 5.2.2 詳細表示
モーダルで明細行を表示：
- 商品名
- 数量
- 単価
- 金額

#### 5.3.1 入力項目

**ヘッダー情報**
- 販売日（デフォルト当日）
- 顧客（プルダウン選択）

**明細情報（1～5行）**
- 商品（プルダウン選択、価格表示）
- 数量（数値入力）

### 6.1 技術スタック

| 層 | 技術 | バージョン |
|----|------|-----------|
| 言語 | Java | 21 |

| テンプレート | Thymeleaf | - |
| ビルドツール | Maven | - |

### 6.2 エンティティ構造

```
SaleHeader (ヘッダー)
├── id: Long
├── customer: Customer
├── saleDate: LocalDate
├── totalAmount: Integer
└── details: List<Sale> (明細リスト)

Sale (明細)
├── id: Long
├── header: SaleHeader
├── product: Product
├── quantity: Integer
└── salePrice: Integer
```

## 7. インターフェース仕様

### 7.1 URL設計

| パス | メソッド | 説明 |
|------|---------|------|
| / | GET | ダッシュボード |
| /products | GET | 商品一覧 |
| /customers | GET | 顧客一覧 |
| /sales | GET | 販売一覧 |
| /sales/new | GET | 販売登録画面 |
| /sales/create | POST | 販売登録 |
| /sales/{id}/detail | GET | 販売明細（Ajax用） |
| /sales/{id}/delete | POST | 販売削除 |

### 7.2 データ初期化

システム起動時に以下のサンプルデータを自動生成する：

- カテゴリ：5種類（小説、ビジネス書、コミック、雑誌、教育・参考書）
- 商品：10件（書籍関連）
- 顧客：5件
- 販売ヘッダー：15件
- 販売明細：30件以上
